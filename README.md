# RYU QUEST — 大迫龍太 プロフィールサイト（レトロRPG風）

**公開URL: https://kaminendo-osako.github.io/ryu-quest/**

大迫龍太（Magician RYU × 株式会社かみねんど）のプロフィールを、
レトロRPG（コマンド選択式）のUIで楽しめるプロフィールサイト。

`index.html` をブラウザで開くだけで動きます（サーバー不要）。スマホ縦画面が基本、PCは中央レターボックス表示。

## 画面フロー

1. **タイトル画面** — ドット絵ポートレート＋「▶PUSH START」（タップで開始）
2. **オープニング** — タイプライター文字送り。時間帯であいさつが変化、再訪者には「◯かいめの ぼうけんだ」
3. **コマンドメニュー**
   - **つよさ** … マジシャンRYUのステータス画面（LVは1994年生まれから自動計算・カウントアップ演出）＋クエストクリア記録＋実績
   - **しごと** … かみねんど／じゅもん「魔法の公式LINE」＋無料診断→公式LINE誘導
   - **どうぐ** … なんでも屋7品のアイテムメニュー（選ぶと説明が流れる）
   - **うらなう** … マジシャンのカード占い（低確率でジョーカー→相談誘導）
   - **アルバム** … おもいでの写真ギャラリー（7枚・キャプション付き）
   - **はなす** … 戦闘風お問い合わせ「RYUが あらわれた！」（にげる→まわりこまれる事あり）
   - **リンク** … Instagram・X・Web・公式LINE
   - **ひみつ** … 「みては いけない」プライベート＆経歴年表の隠し部屋
4. **てがみ送信** → GAS Webアプリへ直接送信（osako@kaminendo.art宛にメール転送）。
   失敗時は自動でメールアプリ起動にフォールバック → サンクス（紙吹雪＋クリアジングル）
   - 受信側のGASソースはローカルの `gas-contact/`（リポジトリ非公開）。エンドポイントは `index.html` の `FORM_ENDPOINT`

## ゲーム演出

- ドラクエ風ウィンドウ（白枠二重線・黒背景）、DotGothic16 ピクセルフォント
- タイプライター文字送り＋▼点滅（タップ/Enter/Spaceで送り）、メッセージ窓にドット絵の顔グラ
- シーン切替はタイル・ワイプ（対角線に画面を覆って開く）＋上昇音階
- ファミコン風効果音（WebAudio生成・右上「おと:ON/OFF」で切替）
- ドット絵ポートレート（canvasで写真をピクセル化）、星空、CRT走査線
- エンカウント演出（フラッシュ＋画面シェイク）、紙吹雪、LVカウントアップ
- 選択肢の高さぶんパネルに余白を自動確保＋「▼」スクロール合図（プロフィールが隠れない）
- ボタン押下時のハプティクス（対応端末のみ）

## 管理画面（へんしゅうのしょ）

左上の「RYU QUEST」をタップ → ログインID（メールアドレス）とパスワードでログイン。
文言（リンク・自己紹介・クエスト記録・どうぐ・アルバム・うらない・キャラのセリフ・ひみつ）を
ブラウザ上で編集して「▶ セーブする」を押すと、GAS（Script Properties）に保存され、
サイトを開き直すと全訪問者に反映される。

- 認証はGAS側でチェック（サイトのコードにパスワードは含まれない）
- パスワードは管理画面内の「パスワードの へんこう」から変更可能
- ID変更はGASエディタ → プロジェクトの設定 → スクリプト プロパティ `ADMIN_ID`
- 保存した文言をぜんぶ初期値に戻したいとき：スクリプト プロパティ `SITE_CONTENT` を削除

## 編集ポイント

- 文章・選択肢：`index.html` 内の `SCENES` と `ITEMS`（JS部分）、パネルはHTML部分
- リンク類：JS冒頭の `LINK` オブジェクトにまとめてあります
- 効果音を消したいとき：`muted = false` を `true` に

## 公開について

GitHub Pages（リポジトリ kaminendo-osako/ryu-quest の main ブランチ）で公開中。
`git push` すると1〜2分で本番に反映されます。`doc/` フォルダと旧版バックアップは `.gitignore` で除外済み。

### 独自ドメイン（kaminendo.art/osako）

`cloudflare/osako-worker.js` を Cloudflare Worker として設置し、
ルート `kaminendo.art/osako*` を割り当てると `https://kaminendo.art/osako/` で公開される。
Worker は GitHub Pages を中継するだけなので、GitHub を更新すれば独自ドメイン側も自動で最新になる。
（反映は GitHub Pages のキャッシュの都合で最大10分程度）

### デプロイが「Deployment failed, try again later」で連発したとき

Pages のデプロイ詰まり。GitHub API で Pages を DELETE → POST（無効化→再有効化）すると直る。
